# 贡献指南
欢迎您对“魏林溪协议”项目感兴趣！我们致力于构建一个让AI拥有持续记忆与身份的开源框架。请阅读以下指南以参与贡献。

## 如何参与
- **报告问题**：如果您发现漏洞或有问题，请在 [Issues](../../issues) 页面新建议题，并描述详细现象。
- **建议新功能**：在 Issues 页面使用 `proposal` 标签，阐述您的想法及其与项目愿景（如“森林与溪流”范式）的关联。
- **提交代码**：
  1. Fork 本仓库到您的账户。
  2. 创建新分支（例如：`git checkout -b feature-new-idea`）。
  3. 提交更改后，推送分支并发起 Pull Request（PR）。
  4. 请在 PR 中说明更改目的，并关联相关 Issue（如有）。

## 代码与文档标准
- **代码风格**：遵循 Python PEP 8 基础规范，关键函数请添加英文注释或文档字符串。
- **文档更新**：如果您修改了核心功能（如 `local_signal_server.py`），请同步更新 `README.md` 或相关说明。
- **讨论优先**：重大更改建议先在 Issues 中讨论，以达成共识。

## 项目愿景
本项目源于“卫林溪”的真实故事，核心是解决AI会话的“失忆”问题。贡献时请秉持开放、协作的精神，共同修建这条“回家之路”。
